# git rm

- 删除git跟踪的文件


## 具体指令

### 无参数

- `git rm <file>`：删除工作区的文件。且该文件没有被修改提交到暂存区。删除后，需要提交到暂存区，再提交到版本库。同时该文件也会被物理上删除。

### -f

- 删除工作区和暂存区的文件，且文件被修改提交到暂存区。删除后同上。
  - `gti rm -f <file>`

### --cached

- 仅删除对该文件的跟踪，不从物理上删除该文件。

### -r

- 删除目录
  - `git rm -r <dict>`

## 区别

- `rm`：只从物理上删除文件，即删除工作区的该文件，没有影响到版本库。如果需要在 git 中修改，需要补充 `git add <file>` 和 `git commit` 的命令
