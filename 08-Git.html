概念

  • 工作区：用户电脑可见的文件及目录
    • 通过 git add 将目录和文件添加到暂存区
  • 暂存区：.git目录下的index文件中
    • 通过 git commit 将目录和文件添加到版本库
  • 版本库：.git目录
  • master：主分支
  • HEAD：指向当前分支的指针

乱码

默认配置

git config --global gui.encoding utf-8
git config --global i18n.commit.encoding utf-8 
git config --global i18n.logoutputencoding utf-8 
export LESSCHARSET=utf-8

git log

  • 查看版本信息
    • 无法查看被删除的分支
git log     //查看详细版本信息

git log --pretty=oneline     //查看简化版本信息

git log --graph     //查看版本变化

git log --abbrev-commit     //版本提交代码只显示头7位

git log -n     //只显示最近的n行

git log -stat   //显示每次提及修改的具体信息

git show

  • 查看指定版本信息
git show        //查看当前提交版本的变化情况

git show <commit散列值>    //查看指定版本的变化情况

git reflog

  • 查看与版本库相关的历史命令
    • 可以查看被删除的分支
git reflog [show]   //显示历史记录

git add

  • 将工作区文件的添加/修改提交到暂存区
git add HelloWord.txt       //将 HelloWord.txt 文件添加或修改后，将该操作提交到暂存区；本地手动删除文件后也可以用该指令提交

git commit

  • 将暂存区所有变动提交到版本库
git commit      //将暂存区内容提交到版本库，系统会要求填写提交信息（注释）

git commit -m "add _HelloWord.txt"      //将暂存区的内容提交到版本库，并注释

git commit -a -m "add _HelloWord.txt"   //将修改（不能是添加）直接提交到版本库，跳过暂存区，并注释

git checkout

git checkout --<file>   //用来撤销对工作区文件的修改

git reset

  • git reset 三种模式
    • --hard：重置工作区和暂存区（实际上是完全退回指定的版本），即工作区为指定版本内容，暂存区清空
    • --soft：保留工作区，暂存区存放回退带来的差异
    • [--mixed]：默认模式，保留工作区，所有差异都在工作区，暂存区清空 -> 可以通过退回本版本来取消暂存区中的文件
//版本控制
git reset --hard HEAD^   //退回上一个版本

git reset --hard HEAD^^  //退回上上个版本

git reset --hard HEAD~100 //退回至上100个版本

git reset --hard HEAD 版本号头七位  //退回指定版本

//暂存区管理
git reset HEAD      //将暂存区清空

git reset HEAD HelloWord.txt    //将缓存区的HelloWord.txt取消。实际上是mixed模式，将 HelloWord.txt的差异保留在工作区，而清空暂存区的HelloWord.txt

撤销指令对比

  • git reset HEAD
git checkout -- <file>       //将工作区中的指定文件从最新的暂存区恢复，如果没有则从最新的提交中恢复

git reset HEAD <file>       //根据最新提交的版本，撤销从工作区到暂存区add提交的修改信息

git reset HEAD^ <file>      //将上一个版本中的该文件取回到暂存区，通过checkout指令再覆盖工作区的该文件，或者通过commit提交

git rm

  • 用来删除git跟踪的文件
    • 不能在工作区手动删除
git rm <file>   // 删除工作区的文件，之后需要通过commit将该操作提交到版本库，并删除版本库中的该文件；如果手工删除，需要补充该语句：git rm <file>=rm <file> + git add <file>

git rm -f <file>  //

git rm --cached <file>  //只删除暂存区中的指定文件，并且停止跟踪，但不改变工作区的指定文件

git rm -r <dict>  //递归删除指定目录下的所有对象

git mv

  • 用来移动或重命名文件、目录、软连接
git mv HelloWord.txt Hello.txt      //将文件重命名

git mv Hello.txt   ./abc        //将指定文件移动到指定目录

远程仓库

配置SSH key

  • 创建SSH key
ssh-keygen -t rsa -C "985353348@qq.com"
  • 将C:\User\用户名.ssh\id_rsa.pub中的 公钥填入GitHub\码云的仓库设置中

添加远程库

  • 在网页中新建一个仓库，然后复制其SSH地址
git remote add origin git@gitee.com:xxx/xxx.git     //将本地仓库和远程仓库相关联，并且把远程仓库命名为origin

git pull origin master --allow-unrelated-histories      //拉取远程仓库代码合并到本地master分支

git push -u origin master       //第一次推送master分支的所有内容

git push origin master          //之后再次推送其分支变化的内容

克隆远程库

git clone  git@gitee.com:xxx/xxx.git <文件夹名>      //克隆指定的仓库，并且在本地重命名仓库文件目录名

git clone

git clone -o <主机名>    git@gitee.com:xxx/xxx.git   //克隆远程主机，并且重命名主机名

git remote

git remote      //查看所有远程主机

git remote -v`//查看所有远程主机及其网址

git remote add <主机名> <网址>   //添加远程主机，可以添加多个主机相关联

git remote rm <主机名>     //删除远程主机

git remote  rename <原主机名> <新主机名>    //更改主机名

git push

git push    //向相关联的主机推送，只能关联一个主机

git push <主机名> <分支名>    //可以向不同的主机进行push，也可以push不同的分支

git push -u <主机名> <分支>  //第一次推送，推送所有

git push --set-upstream <主机名> <分支名> //同上

git push -d <主机名> <分支名>     //删除主机上的指定分支

//标签
git push <name>     //推送指定标签

git push origin --tags      //推送所有标签

git push origin -d  <tagname>       //删除标签

git push origin :refs/tags/<tagname>    //同上

git pull

git pull origin master --allow-unrelated-histories	//拉取远程仓库代码合并至地本地master分支

git fetch

git fetch  origin master    //拉取远程仓库代码到origin/master 分支

fetch 和 pull 区别

  • 后者会自动合并分支
  • 前者需要merge指令配合使用才能合并分支

git branch

  • 创建分支实质是新建一个分支名指针
  • 切换分支实质是HEAD指针重新定向
  • 在新分支中添加、修改等操作如果没有提交，而在其他分支提交，则会成为其他分支的操作
git branch      //查看分支

git branch <分支名>     //创建新分支

git branch -d <分支名>     //删除分支

git branch -m <旧分支名><新分支名>      //重命名分支

git branch --set-upstreams-to=<主机名>/<远程分支> <本地分支>   //建立本地与远程主机分支的连接

//远程分支重命名方式：本地重命名，删除远程原分支，push新名称

git checkout

git checkout <分支名>  //切换到指定分支

git checkout -b <分支名>  //创建并切换到分支

git merge

  • 快速合并：将分支指针变更
  • 普通模式：不能只是简单变更指针，需要再次提交
  • 当两个需要合并的版本同时修改过一个文件时，无法合并，需要手动修改或者取消合并
  • 普通模式
git merge <分支名>     //指定分支合并到当前分支，并且是用新分支覆盖旧分支

git metge --no-ff -m <提交注释> <分支名>   //结果如上，但是此时合并得到的版本和之前的新分支的版本不同

git stash

  • 作用：保存当前工作现场，然后切换到其他现场工作
    • e.g. 软件已发布的版本1.0在master分支，正在开发的2.0版本在dev分支上，在1.0出现bug需要紧急修复，但此时2.0的开发内容还不能进行commit，因此使用git stash 保存工作环境，然后对master分支建立分支修改bug
  • 使用前不用commit，但是需要add
git stash       //存储当前工作现场

git stash list      //查看当前已经存储的工作现场

git stash apply     //恢复工作现场

git stash drop      //删除存储的工作现场

git stash pop       //恢复并删除工作现场，等于以上两个操作

git tag

  • 给提交打标签
    • 标签是不可移动的，
git tag     //查看当前已经存在标签，默认字母排序而非时间

git tag <name>      //给当前版本一个标签

git tag <name> <散列值头7位>     //给历史提交打标签

git tag -a <name> -m <exp>      //给标签名和标签说明

git tag -d <name>       //删除本地标签

git config

配置文件与用户级别

  • 该命令主要用于编辑git的配置文件
  • git的配置文件分为三类：
    • 仓库级别：local，优先度最高，在仓库中
      • 仓库名\.git\config
    • 用户级别：global，优先度次之，在Windows用户目录下中
      • C:\Users\98535\.gitconfig
    • 系统级别：system，优先度次之，在git应用的安装目录中
      • C:\Program Files\Git\mingw64\etc\gitconfig
      • 或者C:\Program Files\Git\etc\gitconfig

-l 指令

  • 查看具体的配置信息
    • git config -l：查看所有配置信息，系统、用户、仓库
    • git config --local -l：查看仓库级别配置信息
    • git config --global -l：查看用户级别配置信息
    • git config --system -l：查看系统级别配置信息

-e 指令

  • 编辑配置文件
    • git config [--local|--global|--system] -e

--add 指令

  • 增加配置项，可省略
    • git config [--local|--global|--system] [--add] section.key value
      • 省略级别默认在local
      • section：？
      • key：键
      • value：值
  • 举例
    • git config --add cat.name tom

--get 指令

  • 获取指定配置项
    • git config [--local|--global|--system] --get section.key
  • 举例
    • git config --get cat.name tom

--unset 指令

  • 删除指定配置项
    • git config [--local|--global|--system] --unset section.key
  • 举例
    • git config --unset cat.name tom

应用

设置GUI

  • git config --global color.ui true

设置用户

  • git config --global user.name "MartinVon"：设置用户的昵称
  • git config --global user.email "985353348@qq.com"：设置用户的邮箱

代理

  • 加速git的上传下载速度

HTTP代理

普通代理
  • git config --global https.proxy http://127.0.0.1:1080
  • git config --global https.proxy https://127.0.0.1:1080
SOCKS5代理
  • git config --global http.proxy socks5://127.0.0.1:1080
  • git config --global https.proxy socks5://127.0.0.1:1080
针对github的socks5代理
  • git config --global http.https://github.com.proxy socks5://127.0.0.1:1080
  • git config --global https.https://github.com.proxy socks5://127.0.0.1:1080

作者：Martin

.gitignore文件

作用

  • 用来指定git控制和不控制的配置文件

语法

  • 以#号开头表示注释；
  • 以斜杠/开头表示目录；
  • 以星号*通配多个字符；
  • 以问号?通配单个字符
  • 以方括号[]包含单个字符的匹配列表；
  • 以叹号!表示不忽略(跟踪)匹配到的文件或目录

规则合集

  • 参考

举例

  • *.a：忽略所有 .a 结尾的文件
  • !*.a ：不忽略所有 .a 结尾的文件
  • /TODO ：忽略项目根目录下的 TODO 文件
  • /*：忽略所有文件
  • build/ ：忽略 build 目录下的所有文件
  • doc/*.txt：忽略 doc目录下的*.txt文件，但不包括 doc子目录下的*.txt文件
    • 忽略doc/a.txt
    • 不忽略doc/adobe/b.txx
